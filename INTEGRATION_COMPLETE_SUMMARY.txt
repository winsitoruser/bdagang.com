COMPLETE WORKFLOW
================================================================================

END-TO-END USER FLOW:

1. USER VIEWS REQUISITIONS
   URL: http://localhost:3001/hq/requisitions
   Action: See list of requisitions
   Result: Table displays with filters

2. USER OPENS REQUISITION DETAIL
   Action: Click on requisition row
   Result: Detail modal opens

3. USER ASSIGNS FULFILLING BRANCH
   Condition: fulfillingBranch is null
   Action: Click "Tentukan Pemenuhan"
   Result: FulfillmentAssignmentModal opens
   
   Sub-flow:
   - Select branch from list
   - Set delivery date (optional)
   - Add notes (optional)
   - Click "Simpan Pemenuhan"
   - Modal closes
   - UI updates with assigned branch

4. USER APPROVES REQUISITION
   Condition: Status is "submitted" or "under_review"
   Action: Click "Setujui" button
   Result: Status changes to "approved"

5. USER CREATES SHIPMENT
   Condition: Status is "approved" AND has fulfilling branch
   Action: Click "Buat Shipment"
   Result: CreateShipmentModal opens
   
   Sub-flow:
   - Select carrier (Internal/JNE/JNT/etc)
   - Enter driver details (if internal)
   - Set delivery date
   - Enter package info (packages, weight, volume)
   - Enter costs (shipping, insurance)
   - Add delivery instructions
   - Add notes
   - Click "Buat Shipment"
   - API creates shipment
   - Success message shows shipment number
   - Modal closes

6. USER TRACKS SHIPMENT
   Condition: Status is "in_transit" or "delivered"
   Action: Click "Lacak Pengiriman"
   Result: ShipmentTrackingModal opens
   
   Display:
   - Shipment number & tracking number
   - Route visualization (from → to)
   - Current status with icon
   - ETA date
   - Tracking timeline with updates:
     * Picked up
     * In transit
     * At hub
     * Out for delivery
     * Delivered
   - Package details (packages, weight)
   - Carrier info (carrier, driver, phone, vehicle)
   - Notes

7. USER EXPORTS DATA
   Action: Click "Export" button
   Result: CSV file downloads with all requisition data
