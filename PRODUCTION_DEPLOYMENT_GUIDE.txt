Security Notes:
- NEVER commit .env to git
- Use strong random passwords (min 32 characters)
- Rotate secrets regularly
- Use environment-specific secrets

10. ROLLBACK PROCEDURES
================================================================================

If Deployment Fails:
----------------------------------
1. Stop the application
   pm2 stop bedagang-admin

2. Restore previous version
   git checkout <previous-tag>
   npm install
   npm run build

3. Rollback database if needed
   npm run db:migrate:undo

4. Restart application
   pm2 restart bedagang-admin

5. Verify rollback successful
   node scripts/test-admin-api.js

Database Rollback:
----------------------------------
# Undo last migration
npm run db:migrate:undo

# Undo all migrations (DANGEROUS)
npm run db:migrate:undo:all

# Restore from backup
psql -U bedagang_user -d bedagang_production < backup.sql

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:
----------------------------------
1. "Failed to fetch module" error
   → Fixed: Role checking standardized across all endpoints

2. Database connection errors
   → Check DB_HOST, DB_PORT, credentials in .env
   → Verify database is running and accessible

3. 401 Unauthorized errors
   → Verify NextAuth session is valid
   → Check NEXTAUTH_SECRET is set
   → Clear browser cookies and re-login

4. 403 Forbidden errors
   → Verify user has admin/super_admin role
   → Check role format in database (case-insensitive now)

5. Slow API responses
   → Check database indexes
   → Review query complexity
   → Monitor connection pool

Logs Location:
----------------------------------
- Application logs: pm2 logs bedagang-admin
- Error logs: ~/.pm2/logs/bedagang-admin-error.log
- Access logs: ~/.pm2/logs/bedagang-admin-out.log
