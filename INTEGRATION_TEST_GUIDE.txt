STEP-BY-STEP TESTING PROCEDURE
================================================================================

TEST 1: FULFILLMENT ASSIGNMENT
-------------------------------
1. Start development server:
   npm run dev

2. Navigate to: http://localhost:3001/hq/requisitions

3. Click on requisition with status "submitted" or "under_review"

4. In detail modal, look for "Dipenuhi Oleh" section

5. If "Belum ditentukan", click "Tentukan Pemenuhan" button

6. Modal should open showing:
   - Available branches list
   - Radio selection UI
   - Delivery date picker
   - Notes textarea

7. Select a branch (e.g., Gudang Pusat Cikarang)

8. Click "Simpan Pemenuhan"

9. Verify:
   ✓ Modal closes
   ✓ Success message appears
   ✓ "Dipenuhi Oleh" now shows selected branch
   ✓ Button changes to "Ubah Pemenuhan"

EXPECTED RESULT: ✅ Fulfillment assignment working


TEST 2: SHIPMENT CREATION
--------------------------
1. Find requisition with:
   - Status: "approved"
   - Has fulfilling branch assigned

2. Click to open detail modal

3. Look for blue box with "Siap untuk Pengiriman"

4. Click "Buat Shipment" button

5. CreateShipmentModal should open showing:
   - Requisition info (from → to)
   - Carrier selection (6 options)
   - Driver details form
   - Shipping details inputs
   - Cost inputs
   - Instructions textarea

6. Fill the form:
   - Select carrier: "JNE"
   - Driver name: "Test Driver"
   - Driver phone: "0812-1234-5678"
   - Vehicle: "B 9999 XYZ"
   - Delivery date: Tomorrow's date
   - Packages: 2
   - Weight: 30 kg
   - Shipping cost: 100000
   - Instructions: "Call before arrival"

7. Click "Buat Shipment"

8. Verify:
   ✓ Loading state shows
   ✓ API call succeeds
   ✓ Success alert appears with shipment number
   ✓ Modal closes
   ✓ Page refreshes

EXPECTED RESULT: ✅ Shipment creation working


TEST 3: SHIPMENT TRACKING
--------------------------
1. Find requisition with status: "in_transit" or "processing"
   (Or manually change mock data status)

2. Click to open detail modal

3. Look for green box with "Shipment Aktif"

4. Click "Lacak Pengiriman" button

5. ShipmentTrackingModal should open showing:
   - Shipment number and tracking number
   - Route visualization (from → to)
   - Current status with icon
   - ETA date
   - Tracking timeline with updates
   - Package details
   - Carrier information

6. Verify timeline shows:
   ✓ Multiple tracking updates
   ✓ Icons for each status
   ✓ Timestamps
   ✓ Locations
   ✓ Descriptions

7. Click "Tutup" to close modal

EXPECTED RESULT: ✅ Tracking modal working


TEST 4: API ENDPOINTS
---------------------
Using browser console or Postman:

A. Test Create Shipment API:
   POST http://localhost:3001/api/hq/requisitions/1/create-shipment
   Headers: 
   - Content-Type: application/json
   - Cookie: (your session cookie)
   
   Body:
   {
     "carrier": "JNE",
     "driverName": "Test Driver",
     "driverPhone": "0812-1234-5678",
     "vehicleNumber": "B 9999 XYZ",
     "estimatedDeliveryDate": "2026-02-27",
     "totalPackages": 2,
     "totalWeight": 30,
     "shippingCost": 100000,
     "deliveryInstructions": "Call before arrival",
     "shippedFromBranchId": "6",
     "shippedToBranchId": "2"

   Expected Response:
   {
     "success": true,
     "message": "Shipment created successfully",
     "data": {
       "id": "ship-...",
       "shipmentNumber": "SH-...",
       ...
     }
   }

B. Test Tracking API:
   GET http://localhost:3001/api/shipments/ship-001/tracking

   Expected Response:
   {
     "success": true,
     "data": {
       "shipment": { ... },
       "trackingUpdates": [ ... ]
     }
   }

C. Test Add Tracking Update:
   POST http://localhost:3001/api/shipments/ship-001/tracking
   Body:
   {
     "status": "delivered",
     "location": "Cabang Bandung",
     "description": "Package delivered successfully"
   }

   Expected Response:
   {
     "success": true,
     "message": "Tracking update added successfully",
     "data": { ... }
   }

EXPECTED RESULT: ✅ All APIs responding correctly

================================================================================
COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE 1: Modal doesn't open
SOLUTION: Check console for errors, verify import paths

ISSUE 2: API returns 401 Unauthorized
SOLUTION: Ensure user is logged in, check session

ISSUE 3: API returns 404
SOLUTION: Verify API route exists, check file paths

ISSUE 4: Button doesn't appear
SOLUTION: Check conditional rendering logic, verify requisition status

ISSUE 5: Form submission fails
SOLUTION: Check required fields, verify API endpoint, check network tab

ISSUE 6: Tracking data not showing
SOLUTION: Verify shipmentId is correct, check mock data

ISSUE 7: Export doesn't download
SOLUTION: Check API response, verify blob creation, check browser settings
