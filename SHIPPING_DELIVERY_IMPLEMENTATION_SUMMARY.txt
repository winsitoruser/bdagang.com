WORKFLOW DESIGN
================================================================================

COMPLETE SHIPPING FLOW:

STEP 1: CREATE SHIPMENT (HQ)
- Requisition approved
- HQ creates shipment
- Assigns carrier/driver
- Sets delivery date
- Assigns items to packages
- Status: pending

STEP 2: PREPARE SHIPMENT (Fulfilling Branch)
- Receives notification
- Picks items from inventory
- Packs into packages
- Generates documents (packing list, delivery note, label)
- Uploads documents
- Marks ready to ship
- Status: ready_to_ship

STEP 3: SHIP ITEMS (Fulfilling Branch/Driver)
- Driver picks up packages
- Enters tracking number
- Confirms shipped time
- Initial tracking update
- Status: in_transit

STEP 4: TRACK SHIPMENT (All Users)
- Real-time location updates
- View tracking history
- Check ETA
- Receive notifications
- Monitor delays

STEP 5: RECEIVE & CONFIRM (Requesting Branch)
- Driver arrives
- Branch verifies items
- Counts quantities
- Checks condition
- Reports discrepancies
- Captures signature & photos
- Confirms delivery
- Status: delivered

STEP 6: COMPLETE (System)
- All items confirmed
- Inventory updated
- Requisition completed
- Documents archived
- Analytics updated

API ENDPOINTS DESIGNED
================================================================================

10 COMPREHENSIVE ENDPOINTS:

SHIPMENT MANAGEMENT:
1. POST /api/hq/requisitions/:id/create-shipment
   - Create new shipment
   - Assign carrier and driver
   - Set delivery details

2. PUT /api/hq/requisitions/:id/shipments/:shipmentId/mark-ready
   - Mark shipment ready to ship
   - Upload documents

3. PUT /api/hq/requisitions/:id/shipments/:shipmentId/ship
   - Mark as shipped/in transit
   - Add tracking number

TRACKING:
4. GET /api/shipments/:shipmentId/tracking
   - Get tracking history
   - View all updates

5. POST /api/shipments/:shipmentId/tracking
   - Add tracking update
   - Update location/status

DELIVERY:
6. POST /api/shipments/:shipmentId/confirm-delivery
   - Confirm delivery
   - Verify items
   - Report discrepancies

MONITORING:
7. GET /api/hq/shipments/active
   - Get all active shipments
   - Filter by status/branch

8. GET /api/hq/shipments/analytics
   - Get shipping metrics
   - Performance KPIs

BRANCH:
9. GET /api/branch/requisitions/:id/shipment
   - Get shipment for branch requisition

10. POST /api/branch/shipments/:shipmentId/receive
    - Branch receives delivery
    - Confirm receipt
