MODULE 1: ROUTE MANAGEMENT (RITASI SYSTEM)
================================================================================

PURPOSE:
Manage delivery routes, rotations, and route optimization for fleet operations.

FEATURES:
1. Route Creation & Management
2. Route Assignment to Vehicles/Drivers
3. Route Optimization
4. Multi-stop Route Planning
5. Route Templates
6. Route History
7. Route Performance Analytics

DATABASE SCHEMA:

Table: fleet_routes
- id (UUID, PK)
- tenant_id (UUID, FK)
- route_number (VARCHAR, unique)
- route_name (VARCHAR)
- route_type (ENUM: delivery, pickup, round_trip, multi_stop)
- start_location (VARCHAR)
- end_location (VARCHAR)
- total_distance_km (DECIMAL)
- estimated_duration_minutes (INTEGER)
- stops (JSONB) - Array of stop locations
- status (ENUM: active, inactive, archived)
- created_at, updated_at

Table: fleet_route_assignments
- id (UUID, PK)
- route_id (UUID, FK)
- vehicle_id (UUID, FK)
- driver_id (UUID, FK)
- shipment_id (UUID, FK, nullable)
- scheduled_date (DATE)
- scheduled_start_time (TIME)
- actual_start_time (TIMESTAMP)
- actual_end_time (TIMESTAMP)
- status (ENUM: scheduled, in_progress, completed, cancelled)
- total_distance_km (DECIMAL)
- fuel_consumed_liters (DECIMAL)
- notes (TEXT)
- created_at, updated_at

Table: fleet_route_stops
- id (UUID, PK)
- route_assignment_id (UUID, FK)
- stop_sequence (INTEGER)
- location_name (VARCHAR)
- location_address (TEXT)
- latitude (DECIMAL)
- longitude (DECIMAL)
- planned_arrival_time (TIMESTAMP)
- actual_arrival_time (TIMESTAMP)
- planned_departure_time (TIMESTAMP)
- actual_departure_time (TIMESTAMP)
- status (ENUM: pending, arrived, departed, skipped)
- notes (TEXT)
- created_at, updated_at

UI COMPONENTS:
1. RouteListPage - List all routes
2. RouteFormModal - Create/Edit route
3. RouteDetailPage - View route details
4. RouteAssignmentModal - Assign route to vehicle/driver
5. RouteMapView - Visual route on map
6. RouteOptimizer - Optimize route stops

API ENDPOINTS:
- GET /api/fleet/routes - List routes
- POST /api/fleet/routes - Create route
- GET /api/fleet/routes/:id - Get route details
- PUT /api/fleet/routes/:id - Update route
- DELETE /api/fleet/routes/:id - Delete route
- POST /api/fleet/routes/:id/assign - Assign route
- GET /api/fleet/routes/assignments - List assignments
- PUT /api/fleet/routes/assignments/:id - Update assignment
- POST /api/fleet/routes/optimize - Optimize route

KPIs:
- Total Routes
- Active Routes
- Average Route Distance
- Average Route Duration
- Route Completion Rate
- On-Time Arrival Rate
- Route Efficiency Score
